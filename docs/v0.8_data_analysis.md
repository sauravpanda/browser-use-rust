## Critical findings (chronological)

1. **Calibration was the dominant gap** (v0.8.9 fix). Upstream's secret was `web_search` fallback when blocked + honest `success=False`. Our agent was hedging "I was unable to access X but here's typical content" with `success=True` on ~10-20 tasks per run.

2. **v0.8.10's prompt addition** ("switch tools or navigate elsewhere") caused over-rotation. v0.8.13 surgical revert recovered most of the loss.

3. **v0.8.17's KEEP_ATTRS expansion** (22 extra attrs) bloated the DOM serialization on dense pages → -3pp. v0.8.21 revert recovered it.

4. **v0.8.18's PIVOT**: enabling Gemini-3 `max_output_tokens=8096` doubled per-call output (1522 → 4037 tokens). Net effect: HALF the steps per task, HALF the cost. Best efficiency change of the line.

5. **`thinking_level` was being silently passed as raw lowercase string** (v0.8.22 fix). Eval framework passes "medium"; we forwarded raw string instead of `gtypes.ThinkingLevel.MEDIUM` enum. Pydantic might coerce; might not. Now explicit.

6. **`CancelledError` was slipping past every defense** (v0.8.23 fix). The "Stage errors: run_agent:" pattern we'd been hunting since v0.8.3 was eval-framework cancellation propagating through `session.screenshot()` (or any awaiting CDP call) when the eval's `wait_for(stage_func, timeout)` fired. Python 3.8+ moved `asyncio.CancelledError` to inherit `BaseException` not `Exception`, so our `except Exception` did NOT catch it. Fixed in v0.8.23 with explicit `except asyncio.CancelledError` + per-CDP-op `asyncio.wait_for` caps (5s url, 15s screenshot, 30s dom_snapshot).
